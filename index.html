🔭 Photon Requirement Calculator Static GitHub Pages app powered by Pyodide
Enter a list of target SNR values and the page solves for the total detected photons needed to reach each target in two cases: one diffraction-limited injection where all flux lands in a single spaxel, and one seeing-limited IFU case where the same total flux is spread across many spaxels before summing. The same target SNR list is applied to both scenarios so the comparison is always done at identical requested SNR values. The Python code runs entirely in the browser, so it works on GitHub Pages without a backend.

✨ Diffraction-limited Injection Scenario

Gain is assumed to be 1, so photons and electrons are treated identically here. The outputs report total detected photons across the full stack, plus the photon penalty for the seeing-limited IFU case relative to the diffraction-limited injection case.

📐 Inverse Noise Model

The calculator solves for the total signal $S$ needed to reach a requested SNR after stacking $N$ frames. With one read per frame and gain $= 1$, the total variance for a scenario with $n_{\mathrm{pix}}$ extracted pixels is:

$$\mathrm{SNR} = \frac{S}{\sqrt{S + B}}, \qquad B = N\,n_{\mathrm{pix}}\left(\sigma_{\mathrm{RON}}^2 + I_{\mathrm{dark}}\,t\right)$$

Rearranging gives a closed-form solution for the required total photons:

$$S_{\mathrm{req}} = \frac{1}{2}\left[\mathrm{SNR}^2 + \sqrt{\mathrm{SNR}^4 + 4\,\mathrm{SNR}^2\,B}\right]$$
STotal detected photons across the full stack
NNumber of stacked frames
tExposure time per frame in seconds
n_pixTotal extracted detector pixels in the scenario
σ_RONReadout noise per pixel per frame
I_darkDark current in e⁻ / s / pix

Diffraction-limited injection scenario: $n_{\mathrm{pix}} = n_{\mathrm{extract,compact}}$. Seeing-limited IFU scenario: $n_{\mathrm{pix}} = n_{\mathrm{spaxels}} \times n_{\mathrm{extract,extended}}$.
